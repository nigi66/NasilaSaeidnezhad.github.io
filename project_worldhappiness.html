Cleaning the World Happiness Report Dataset with SQL

The World Happiness Report ranks countries based on various indicators like GDP per capita, social support, life expectancy, freedom, generosity, and perceptions of corruption. The dataset is published annually and consists of multiple CSV files, each representing a different year. While rich in insights, the dataset is messy: column names differ slightly between years, there are missing values, country names are sometimes inconsistent, and some numeric fields are stored as strings. In this tutorial, I demonstrate how to clean and standardize this multi-year dataset using MySQL, preparing it for analysis or visualization.

Key Features

  • Importing multiple CSV files into MySQL
  • Handling missing values
  • Standardizing column names across years
  • Removing duplicates
  • Normalizing country names
  • Combining data into a single clean table for analysis
